## ✅ 승인 절차

### 1단계: 신청 확인
- `/admin/bank-transfers` 접속
- "대기중" 카운트 확인
- 신청 목록에서 노란색 "대기중" 배지 확인

### 2단계: 입금 확인
신청 정보 확인:
- 신청자 이름
- 이메일
- 연락처
- 플랜 이름
- 금액
- 신청일시

계좌 입금 내역과 대조:
- 입금자명 = 신청자 이름?
- 입금액 = 신청 금액?
- 입금일 = 신청일?

### 3단계: 승인
1. ✅ "승인" 버튼 클릭 (초록색)
2. 확인 팝업에서 "확인" 클릭
3. 성공 메시지: "✅ 승인되었습니다!"
4. 페이지 자동 새로고침

**자동 처리됨:**
- 사용자 학원(academy) 생성
- 구독(subscription) 활성화 (1개월)
- 사용량 추적 초기화
- 신청 상태 → "승인완료"

---

## ❌ 거절 절차

### 거절이 필요한 경우
- 입금 확인 불가
- 입금액 불일치
- 중복 신청
- 기타 사유

### 절차
1. ❌ "거절" 버튼 클릭 (빨간색)
2. 거절 사유 입력 (예: "입금 확인 불가")
3. "확인" 클릭
4. 성공 메시지: "✅ 거절 처리되었습니다."
5. 페이지 자동 새로고침

---

## 📊 통계 카드 설명

### 대기중 (노란색)
- 아직 처리하지 않은 신청
- **이 숫자가 0이면 처리할 신청이 없음**

### 승인완료 (초록색)
- 승인하고 구독을 활성화한 신청
- 총 승인 건수

### 거절 (빨간색)
- 거절 처리한 신청
- 총 거절 건수

---

## ⚠️ 주의사항

### DO
- ✅ 반드시 입금 확인 후 승인
- ✅ 신청자 정보와 입금 정보 정확히 대조
- ✅ 거절 시 명확한 사유 입력

### DON'T
- ❌ 입금 확인 전 승인 금지
- ❌ 승인 후 취소 불가 (개발자 문의 필요)
- ❌ admin@superplace.co.kr 외 다른 계정으로 승인 불가

---

## 🐛 문제 발생 시

### 페이지가 로드되지 않음
→ `/api/admin/init-payment-tables` 접속하여 데이터베이스 초기화

### 승인이 안 됨
→ 브라우저 콘솔(F12) 확인 후 개발자에게 오류 메시지 전달

### 통계가 안 맞음
→ 페이지 새로고침 (F5)
